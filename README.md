1. Модуль 1: разработка модулей программного обеспечения для компьютерных систем.
1.1 Анализ и проектирование.
Спецификация модуля
Функции модуля: 
– подача заявки на ремонт, то есть сообщение о неисправности автомобиля заказчиком, содержащая данные о виде автомобиля, модели, описании проблемы, личные данные и контактную информацию о клиенте, существует возможность редактирования заявки;
– регистрация заявки, то есть прием и заведение заявки в систему учета оператором, присвоение заявке уникального идентификатора и сохранение информации о заявке;
– обработка заявки, то есть анализ заявки, назначение приоритетности и автомеханика для ремонта, возможна связь между клиентом и автомехаником в процессе ремонта;
– исполнение заявки, то есть процесс ремонта автомобиля, возможно возникновение необходимости заказа автозапчастей, координация работ с другими специалистами;
– отчетность и информирование, то есть фиксация результата выполнения работ, после завершения ремонта автомеханик подает отчет о проделанной работе;
– мониторинг и анализ, то есть контроль и анализ процесса учета заявок на ремонт, включающие отслеживание времени обработки заявок, качество выполнения работ и прочие параметры, позволяющие оптимизировать бизнес-процесс, особый акцент на расчете среднего времени обработки заявки с целью определить, сколько времени занимает обслуживание одного клиента и как можно оптимизировать работу.
Входные данные: содержимое заявки (тип автомобиля, модель, суть проблемы), данные заказчика.
Выходные данные: дата начала заявки, уникальный идентификатор заявки, статус заявки, дата завершения ремонта, отчет о выполнении работы.
